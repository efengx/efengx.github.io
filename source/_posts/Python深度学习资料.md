## 解决机器学习问题的通用模版

```bash
问题定义 -> 评估 -> 特征工程 -> 解决过拟合

1，数据可用性
2，面对的问题类型？（是二分类、多分类问题、标量回归问题、向量回归问题、还是多分类、多标签问题？或者其他问题，比如聚类、生产或强化学习？确定问题类型有助于你选择模型架构、损失函数等）
3，假设输出是可以根据输入进行预测的；假设可用数据包含足够多的信息，足以学习输入和输出之间的关系。（需要注意非平稳问题）(机器学习预测未来存在一个假设，就是未来的规律与过去相同，但事实往往并非如此)
4，衡量成功的指标？要取得成功，就必须给出成功的定义：精度？准确率？召回率？客户保留率？衡量成功的指标将指引你选择损失函数，即模型要优化什么。它应该直接与你的目标（如业务成功）保存一致。平衡分类问题（每个类别的可能性相同），精度和接收者操作特征曲线下面积是常用指标。对于类别不平衡的问题，你可以使用准确率和召回率。对于排序问题或多标签分类，你可以使用平均准确率均值。自定义衡量成功的指标也很常见。（通过 Kaggle 网站上的数据科学竞赛，可以查看各种各样的问题和评估指标）
5，确定评估方法：留出验证集、K折交叉验证、重复的K折验证（大多数情况下，第一种方法足以满足要求。）
6，准备数据？需要将数据格式化为张量，张量的取值通常应该缩放为较小的值，比如在[-1,1]区间或[0,1]区间。如果不同的特征具有不同的取值范围（异质数据），那么应该做数据标准化。你可能需要做特征工程，尤其是对于小数据问题。
7，开发比基准更好的模型？即开发一个小型模型，它能够打败纯随机的基准。在MNIST数字分类的例子中，任何精度大于0.1的模型都可以说具有统计功效；在IMDB的例子中，任何精度大于0.5的模型都可以说具有统计功效。但是不一定总是能获得统计功效，原因可能是问题的答案并不在输入数据中（1，假设输出是可以根据输入进行预测的；2，假设可用的数据包含足够多的信息，足以学习输入和输出之间的关系。）还需要以下三个参数来构建第一个工作模型：最后一层的激活、损失函数、优化配置。
8，扩大模型规模：开发过拟合的模型。以寻找欠拟合和过拟合的界限。开发过拟合很简单只需要，添加更多的层、让每一层变的更大、训练更多的轮次。要始终监控训练损失和验证损失，以及你所关心的指标的训练值和验证值。如果你发现模型在验证数据上的性能开发下降，那么就出现了过拟合。
9，模型正则化与调节超参数：需要不断第调节模型、训练、在验证数据上评估（这里不是测试数据）、再次调节模型，然后重复这一过程，直到模型达到最佳性能。你应该尝试以下几项（添加dropout、尝试不同的架构、增加或减少层数、添加L1和L2正则化、尝试不同的超参数（比如每层的单元个数或优化器的学习率），以找到最佳配置）、反复做特征工程：添加新特征或删除没有信息量的特征。
```

## 卷积神经网络特性

```bash
密集连接层和卷积层的区别，密集连接层从输入特征空间中学到的是全局模式，而卷积层学到的是局部模式。

1. 具有平移不变性。该特性使卷积网络具有泛化效果。真正的3维空间的事物就是可以随意平移。

2. 学到模式的空间层次结构。使得卷积神经网络科技在局部上对事物进行幻化匹配。

卷积核（是一个固定的张量，卷积核比较小）- 提取特征

池化层（降低数据的维度，下采样）-  数据降维，避免过拟合

全连接层（根据不同任务输出我们想要的结果）

```

## 理解卷积神经网络

```bash
1. 将复杂问题简单化，把大量参数降维成少量参数在做处理。

2. 数据增强将会极大降低过拟合。

3. 用预训练的网络做特征提取。对预训练的网络进行微调。

4. 数据增强，从现有样本中生成更多的训练数据。其方法是利用多种能够生成可信图像的随机变换来增加样本。

5. 变化趋势
- 网络中特征图的深度在逐渐增大（从32增大到128）
- 而特征图的尺寸在逐渐减小（从150x150减小到7x7）
```
